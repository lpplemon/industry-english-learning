# 行业英语学习产品落地执行清单

## 0. 最终目标

用户输入行业和产品后，系统持续提供专业单词、例句和情景对话；支持美式/英式发音、学习目标、复习、小游戏、个人内容导入，并可在手机和电脑浏览器中使用。

## 1. 当前可用版本

- [x] 用户自由填写行业和产品
- [x] 本地生成分批词库，每批 5000 条
- [x] 每组 10 条，完成后自动进入下一组
- [x] 单词、中文释义、例句、场景和对话
- [x] 美式/英式浏览器发音
- [x] 掌握、复习和笔记
- [x] 配对、填空、对话互动
- [x] CSV/TSV 内容导入
- [x] 浏览器本地保存进度
- [x] GitHub 公开仓库和 Pages 发布配置

当前版本是可演示 MVP。词库由本地规则生成，不等于真实大模型生成。

## 2. 第一阶段：稳定公开演示版

## 3. 第二阶段：真实 AI 词库服务

GitHub Pages 只能托管前端，不能安全保存 AI 密钥。真实 AI 必须通过后端调用。

### 3.1 建立后端最小项目

- [ ] 新建 `server` 目录
- [ ] 建立 `POST /api/generate-learning-items`
- [ ] 请求参数只包含行业、产品、难度和数量
- [ ] 返回统一 JSON 数据结构
- [ ] 使用环境变量保存 AI 密钥
- [ ] 增加参数校验和错误提示

最小请求：

```json
{
  "industry": "新能源汽车出口",
  "product": "充电桩",
  "level": "intermediate",
  "count": 20
}
```

最小返回：

```json
{
  "items": [
    {
      "word": "charging protocol",
      "meaning": "充电协议",
      "example": "This charger supports the latest charging protocol.",
      "exampleCn": "这款充电器支持最新的充电协议。",
      "scenario": "向海外客户介绍兼容性",
      "dialogue": [
        { "role": "customer", "text": "Which protocol does it support?" },
        { "role": "sales", "text": "It supports both CCS and GB/T." }
      ]
    }
  ],
  "hasMore": true
}
```

验收：前端输入行业和产品后，能获得 20 条不重复、结构完整的 AI 内容。

### 3.2 分批生成，不一次生成 5000 条

- [ ] 后端每次生成 20 至 50 条
- [ ] 数据库记录用户已经获得的词条
- [ ] 下一批请求携带已学主题摘要
- [ ] 后端去重后再返回
- [ ] 显示“专业词库目标 5000”，而不是一次请求 5000
- [ ] 达到 5000 后建立新一轮词库版本

验收：请求速度稳定，重复率低，达到 5000 后仍能继续学习。

## 4. 第三阶段：账号和云端进度

### 4.1 用户账号

- [ ] 支持邮箱验证码或第三方登录
- [ ] 创建用户资料表
- [ ] 行业和产品可以修改

### 4.2 学习记录

- [ ] 多设备同步

验收：同一账号换设备登录后，学习进度一致。

## 5. 第四阶段：记忆与复习算法

- [ ] 为每条内容增加熟练度 0 至 5
- [ ] 答错后缩短复习间隔
- [ ] 连续答对后延长复习间隔
- [ ] 每天优先显示到期复习内容
- [ ] 新内容和复习内容按目标混合
- [ ] 提供“今天为什么学习这些内容”的简短解释

验收：用户每天打开后都有合理的新学和复习任务，不需要手工挑选。

## 6. 第五阶段：导入个人词库

- [ ] 提供 CSV/XLSX 模板下载
- [ ] 导入前预览前 10 行
- [ ] 标出缺失字段和重复内容
- [ ] 用户确认后写入个人词库
- [ ] 缺少例句时由 AI 补充
- [ ] 缺少对话时由 AI 补充

建议列名：`word`、`meaning`、`example`、`scenario`、`dialogue`。

验收：导入 1000 行表格不会卡死，错误行有明确提示。

## 7. 第六阶段：发音和口语互动

- [ ] 保留浏览器美式/英式朗读
- [ ] 显示当前实际使用的语音名称
- [ ] 找不到目标语音时提示用户安装语音包
- [ ] 增加语速控制
- [ ] 增加跟读录音
- [ ] 增加语音识别结果
- [ ] AI 对发音和表达给出反馈

验收：用户可以听、跟读、查看识别文本，并获得可理解的改进建议。

## 8. 第七阶段：互动小游戏

- [ ] 错题自动进入复习队列

验收：游戏结果会改变熟练度，而不是只显示分数。

## 9. 第八阶段：每日目标和激励

- [ ] 展示今日完成比例
- [ ] 目标过高时给出合理建议
- [ ] 连续学习天数
- [ ] 允许补学，但不制造惩罚感
- [ ] 每周学习报告

## 11. 推荐实施顺序

1. 让 GitHub Pages 演示版成功上线。
2. 完成一次完整的人工体验测试。
3. 搭建真实 AI 后端，只生成 20 条做通路验证。
4. 接入数据库和账号。
5. 实现分批生成与去重。
6. 实现记忆复习算法。
7. 完善导入、口语和小游戏。
8. 增加运营数据、额度和生产监控。

## 12. 每一步的工作方式

每次只处理一个任务，并按下面格式推进：

1. 明确本步目标。
2. 修改最少数量的文件。
3. 在本地验证。
4. 给出用户可见的验收方法。
5. 验收通过后再进入下一步。

当前下一步：解决 GitHub Pages 部署排队或失败问题，并验证公开网址。
